2.版本管理的主要功能有：
(1) 集中管理档案，安全授权机制：档案集中地存放在服务器上，经系统管理员授权给各个用户。用户通过check in和check out的方式访问服务器上的文件，未经授权的用户则无法访问服务器上的文件。
(2) 软件版本升级管理：每次登入时，在服务器上都会生成新的版本，任何版本都可以随时检出编辑。
(3) 加锁功能：在文件更新时保护文件，避免不同的用户更改同一文件时发生冲突。
(4) 提供不同版本源程序的比较。

3.版本管理工具：SVN    GIT    TFS
（1）SVN是Subversion的简称，是一个开放源代码的版本控制系统，相较于RCS、CVS，它采用了分支管理系统，它的设计目标就是取代CVS。互联网上很多版本控制服务已从CVS迁移到Subversion。说得简单一点SVN就是用于多个人共同开发同一个项目，共用资源的目的。

（2）TFS（Team Foundation Server）是一个高可扩展、高可用、高性能、面向互联网服务的分布式文件系统，主要针对海量的非结构化数据，它构筑在普通的Linux机器集群上，可为外部提供高可靠和高并发的存储访问。TFS为淘宝提供海量小文件存储，通常文件大小不超过1M，满足了淘宝对小文件存储的需求，被广泛地应用在淘宝各项应用中。它采用了HA架构和平滑扩容，保证了整个文件系统的可用性和扩展性。同时扁平化的数据组织结构，可将文件名映射到文件的物理地址，简化了文件的访问流程，一定程度上为TFS提供了良好的读写性能。

4.git客户端命令
（1）克隆远程仓库至本地：右键Git Bash 将客户端与账户关联， 输入 $ git clone URL/远程仓库的地址
（2）进入本地库 ：$ cd 仓库名
（3）查看本地库的状态：$ git status
（4）新建文件/文件夹后，将文件上传至本地仓库，可指定文件或全部上传（.）：$ git add .
（5）添加备注：   $ git commit -m "add by Joker"
（6）将本地仓库同步至远程仓库：   $ git push
（7）将远程仓库的内容更新同步至本地仓库：$ git pull   （显示Already up-to-date.即为更新同步完成）
